# TC-12.2 Evidence File Upload Verification

Story: 12.2 Evidence File Upload
Date: 2026-06-15

## Scope

Verified evidence upload metadata for the No-CUI MVP, including acknowledgement gating, validation and malware scan usability state, version history, RBAC-protected download/delete metadata access, and audit logging.

## Setup Data

- Tenant IDs: deterministic per test case in `EvidenceFileUploadTests`
- User IDs: deterministic per test case in `EvidenceFileUploadTests`
- Evidence item IDs: deterministic per test case in `EvidenceFileUploadTests`
- Test upload file metadata: PDF content type, 1024 bytes

## Test Cases

### TC-12.2.1 Upload Requires Acknowledgement

- Steps: POST `/api/evidence-items/{id}/upload-intents` before acknowledging the current No-CUI notice.
- Expected: API blocks upload with precondition failure.
- Actual: API returned HTTP 428.
- Result: Passed.

### TC-12.2.2 Usability Gated By Validation And Scan

- Steps: Acknowledge No-CUI notice, create upload intent, request latest file metadata.
- Expected: Upload metadata exists but is not usable until validation and scan state allow it.
- Actual: Validation status was `accepted`, malware scan status was `scan-pending`, and `isUsable` was `false`.
- Result: Passed.

### TC-12.2.3 Version History Preserved

- Steps: Acknowledge No-CUI notice, create two upload intents for the same evidence item.
- Expected: Replacement creates a new version without overwriting previous metadata.
- Actual: Stored versions had version numbers `1` and `2` with original and replacement file names preserved.
- Result: Passed.

### TC-12.2.4 Download/Delete Permissions And Audit

- Steps: Acknowledge No-CUI notice, create upload intent, download latest file metadata, attempt delete with view-only permission, delete with manage permission, inspect audit log.
- Expected: Download and delete honor RBAC, and download/delete actions are audit logged.
- Actual: Download succeeded with `ViewEvidence`; delete with `ViewEvidence` returned HTTP 403; delete with `ManageEvidence` returned HTTP 200; audit log contained `Downloaded` and `Deleted` events for `EvidenceFileVersion`.
- Result: Passed.
