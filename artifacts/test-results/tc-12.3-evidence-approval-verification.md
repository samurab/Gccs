# TC-12.3 Evidence Approval Verification

Story: 12.3 Evidence Approval
Date: 2026-06-15

## Scope

Verified the evidence approval workflow for authorized review decisions, rejection validation, approved-evidence report eligibility, and decision audit logging.

## Setup Data

- Tenant IDs: deterministic per test case in `EvidenceApprovalTests`
- Evidence item IDs: deterministic per approval, rejection, archive, and expire test case
- Report package: completed prime evidence package linked to evidence for approved-evidence inclusion
- Roles: Contributor, Compliance Manager, and Auditor through development auth role headers

## Test Cases

### TC-12.3.1 Authorized Approval Only

- Steps: Attempt approval as Contributor, then approve as Compliance Manager.
- Expected: Contributor is forbidden; configured approval role can approve and stamps approved metadata.
- Actual: Contributor received HTTP 403; Compliance Manager received HTTP 201; evidence status became `Approved` with reviewer metadata.
- Result: Passed.

### TC-12.3.2 Rejection Requires Reason

- Steps: Submit a reject decision with a blank comment.
- Expected: Validation fails and evidence state does not change.
- Actual: API returned HTTP 400 and evidence remained `Submitted`.
- Result: Passed.

### TC-12.3.3 Approved Evidence In Reports

- Steps: Link submitted evidence to a completed prime evidence package, approve it, then read approved evidence packages.
- Expected: Approved evidence appears in the package and includes approval metadata.
- Actual: Report package returned the approved evidence item with `Approved` status and approved timestamp.
- Result: Passed.

### TC-12.3.4 Approval Decision Audit

- Steps: Record approve, reject, archive, and expire decisions for tenant evidence.
- Expected: Each decision is audit logged.
- Actual: Audit log contained `Approved`, `Rejected`, `Archived`, and `Expired` events for the corresponding evidence items.
- Result: Passed.
