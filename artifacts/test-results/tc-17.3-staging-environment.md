## Staging Coverage

- CI/CD deployment: `.github/workflows/staging.yml` builds API and web artifacts, generates an idempotent migration script, validates staging infrastructure coverage, and runs a staging deployment step.
- No production customer data: staging is synthetic-only, No-CUI, and documented to reject production customer data, production secrets, production snapshots, and production customer uploads.
- Health checks: post-deploy smoke tests call `/health` and require API, database, cache, object storage, and background job signals.
- Smoke visibility: staging smoke results are uploaded as `staging-smoke-test-results` so success or failure is visible in CI/CD.
