# TC-13.1 CMMC Level Selection Verification

Story: 13.1 CMMC Level Selection
Date: 2026-06-15

## Scope

Verified the first CMMC readiness workspace slice: Level 1 and Level 2 assessment creation, target status/dates/owner metadata, company and contract links, control-status progress calculation, UI creation flow, and audit logging.

## Setup Data

- Tenant IDs: deterministic per test case in `CmmcAssessmentTests`
- CMMC levels: `Level1` and `Level2`
- Control IDs: `AC.L1-3.1.1` and `IA.L1-3.5.1`
- UI route: `#/cmmc`

## Test Cases

### TC-13.1.1 Create Readiness Assessment

- Steps: Create Level 1 and Level 2 readiness assessments.
- Expected: Target level, status, dates, and responsible owner are stored.
- Actual: Both assessments were created with expected level, `Planned` status, assessment date, affirmation due date, and `Security` owner.
- Result: Passed.

### TC-13.1.2 Link To Profile And Contracts

- Steps: Create an assessment with company profile and contract IDs, then fetch detail.
- Expected: Detail response displays linked company profile and contract IDs.
- Actual: Detail response returned the configured `companyProfileId` and `contractIds`.
- Result: Passed.

### TC-13.1.3 Completion Progress Summary

- Steps: Update two control statuses, one implemented and one partially implemented, then fetch assessment detail.
- Expected: Progress summary recalculates from control statuses.
- Actual: Summary showed total `2`, implemented `1`, partially implemented `1`, and completion percentage `50`.
- Result: Passed.

### TC-13.1.4 Assessment Changes Audited

- Steps: Create an assessment, update its status, and inspect audit logs.
- Expected: Create and update/status-change events are audit logged.
- Actual: Audit log contained `Created` and `Updated` events for the CMMC assessment, with updated status metadata.
- Result: Passed.
